# Task-3: Create and Configure a Virtual Private Cloud (VPC) with Subnets
Setting Up a VPC on AWS:

1. In the search bar, type VPC and select VPC from the results.

2. On the VPC Dashboard, click Create VPC.

3. Choose VPC only or VPC and more based on your requirements: VPC only: Creates a VPC without additional resources. VPC and more: Creates a VPC along with subnets, route tables, and gateways.

4. Provide a Name tag for your VPC.

5. Specify the IPv4 CIDR block (e.g., 10.0.0.0/16) to define the IP address range.

6. (Optional) Enable IPv6 CIDR block if needed.

7. Choose the Tenancy option: Default: Instances share hardware with other AWS accounts. Dedicated: Instances run on hardware dedicated to your account.

8. (Optional) Add tags by entering a key-value pair.

9. Click Create VPC to finalize the setup.

# Adding Subnets:

1. Navigate to Subnets in the VPC Dashboard.

2. Click Create Subnet.

3. Select the VPC you just created.

4. Provide a Name tag, choose an Availability Zone, and specify the IPv4 CIDR block for the subnet.

5. Click Create Subnet.
